# KOSPI200 Quad-Core Bot Secure
본 프로젝트는 기관급 안정성을 목표로 설계된 **24개 핵심 모듈 기반의 자동 매매 시스템**입니다. 결정론적 상태 머신(FSM)과 이벤트 소싱 기반의 아키텍처를 통해, 시스템의 모든 상태를 추적하고 어떠한 장애 상황에서도 즉각적인 복구(Recovery)가 가능하도록 설계되었습니다.
## 🏛️ 시스템 아키텍처
본 시스템은 총 24개의 독립적인 에이전트 모듈이 EventBus를 통해 느슨하게 결합되어 동작합니다.
 * **핵심 제어부 (main.py, event_bus.py)**: 시스템의 심장부로 모든 에이전트의 통신을 중재합니다.
 * **상태 관리 (fsm/):** oms_fsm.py를 통해 주문의 14단계 생애주기를 엄격히 제어합니다.
 * **방어 체계 (guards/):** fat_finger.py, margin_diet.py, sanity_check.py 등이 실시간으로 자산을 보호합니다.
 * **영속성 (event_store.py):** WAL(Write-Ahead Log)을 통해 모든 이벤트를 디스크에 안전하게 기록합니다.
## 🚀 주요 기능 및 방어 기제
 * **결정론적 복구:** replay_events() 기능을 통해 시스템 재기동 시 사고 직전 상태로 복구.
 * **실시간 리스크 차단:** 주문이 전송되기 전, 가격/수량 편차와 증거금 상태를 하드웨어 수준에서 검증.
 * **로그 보안:** 모든 민감 정보는 마스킹 처리되며, 메모리 잔류 데이터는 즉시 제로화(Zeroing).
 * **자동 정합성 검증:** GitHub Actions를 통한 문법 및 구조적 정합성 자동 검사 시스템 구축.
## 🛠️ 모듈 구조 (전체 24종)
 1. main.py - 시스템 통합 사령탑
 2. event_bus.py - 신경망 통신 모듈
 3. data_contract.py - 데이터 표준 헌법 (Dataclass)
 4. fsm/oms_fsm.py - 주문 상태 머신
 5. fsm/feature_flag.py - 비상 정지 기능 스위치
 6. guards/fat_finger.py - 주문 오류 방지 가드
 7. guards/margin_diet.py - 증거금 보호 가드
 8. guards/sanity_check.py - 논리 오류 방지 가드
 9. guards/security.py - 데이터 보안 및 메모리 보호
   ... (총 24개 핵심 모듈 완비)
